Other types of addresses

This is to complement Logical and Physical addresses, which can be seen at week 6 chapter 9 .

Symbolic Addresses

  • Variables declared by the dev
  • int x; Here, x is symbolic as it has no specific memory location yet.
  • They are used to represent memory locations of variables or functions that don't have hard values yet.

Relocatable Adresses

  • Not tied to a specific location, rather can be "relocated" during runtime or during the loading process.
  • For instance, if we have function add(num1, num2) for our code snippet, these would be relocatable as their location would get specified at runtime (I think).
